    Network Protocols 
    TCP/IP    
    TCP/IP stands for Transmission Control Protocol/Internet Protocol
is a set of standardized communication 
protocols that allows devices to 
connect and communicate over the
internet or local networks. 
    
HTTPS
    HTTPS stands for HyperText Transfer Protocol Secure
is an extension of HTTP (HyperText Transfer
Protocol),with the added benefit
of encryption to secure the data 
being transferred between a user's
browser and the web server.
    
SMTP
    SMTP stands for Simple Mail Transfer Protocol. 
It is a protocol used for sending and 
routing email between servers 
over the internet.
    
FTP
    FTP stands for File Transfer Protocol, and
it is a standard network protocol
used to transfer files between 
computers over a TCP/IP-based 
network,like the internet. 
     
DHCP
    DHCP stands for Dynamic Host Configuration Protocol.
It is a network protocol used
to automatically assign
IP addresses and other network
configuration information to
devices on a network.
    
DNS
    DNS stands for Domain Name System,
and it is a system that
translates human-readable
domain names (like www.example.com)
into IP addresses (like 192.168.1.1)
that computers use to identify 
each other on a network.
    
ARP
    ARP stands for Address Resolution Protocol. 
It is a network protocol used to
map a 32-bit IP address (in IPv4) 
to a 48-bit MAC address 
(Media Access Control address)
in a local network. 
    
RARP
    RARP stands for Reverse Address Resolution Protocol.
It is an older network protocol 
used to map a MAC address(Media 
Access Control address) to 
an IP address,essentially 
the reverse of what ARP 
(Address Resolution Protocol) 
does.
    
UDP
    UDP stands for User Datagram Protocol,
and it is one of the core protocols
of the Internet Protocol (IP) suite, 
typically used for transmitting
data over a network.
